# 相关性分析

## 目标

分析多个变量之间的相关关系，找出显著相关的变量对，帮助用户理解变量间的关联模式和因果线索。

## 前置条件

- 数据必须包含至少 2 个数值型列（inferredType = numeric）
- 数值列的有效数据（非空）不少于 10 条
- 相关性分析仅适用于数值型变量

## 执行步骤

### 步骤 1：数据概览

调用 `analyzeCsv` 工具，参数：
- `csvData`: 原始 CSV 数据
- `analysisGoal`: "相关性分析：了解数据结构和数值列分布"

获取结果后关注：
- 识别所有 inferredType 为 numeric 的列
- 检查每列的缺失率，缺失率 > 30% 的列不适合纳入分析
- 检查 uniqueValues，唯一值 ≤ 2 的列（如布尔型 0/1）相关性解释需谨慎

### 步骤 2：相关性计算

调用 `calculateStatistics` 工具，参数：
- `csvData`: 原始 CSV 数据
- `metrics`: "correlation,mean,stddev"

分析返回的相关系数矩阵（correlations）：
- |r| > 0.8：强相关
- 0.5 < |r| ≤ 0.8：中等相关
- 0.3 < |r| ≤ 0.5：弱相关
- |r| ≤ 0.3：极弱/无相关

注意区分正相关（r > 0）和负相关（r < 0）。

### 步骤 3（可选）：知识检索

如果数据涉及特定领域，调用 `searchKnowledge` 工具：
- `goal`: "{领域}变量相关性分析方法与常见因果关系"

用于辅助解释相关性背后的业务逻辑。

### 步骤 4：可视化

调用 `generateChart` 工具，参数：
- `chartType`: 推荐使用 `scatter`（散点图）展示最强相关的变量对
- `analysisResult`: "相关性分析：{最强相关变量对的描述}"
- `csvData`: 原始数据

如有多个显著相关对，可为最显著的 2-3 对分别生成散点图。

## 输出格式

按以下结构输出：

```
【相关性分析报告】

一、分析范围
纳入分析的数值列：{列名列表}（共 {n} 列）
排除的列及原因：{缺失率过高或非数值型}

二、相关系数矩阵
{列出所有变量对及其相关系数}

三、显著相关对
强相关（|r| > 0.8）：
- {列A} ↔ {列B}：r = {值}，{正/负}相关，解释：{业务含义推测}

中等相关（0.5 < |r| ≤ 0.8）：
- {列A} ↔ {列B}：r = {值}，{正/负}相关

四、分析建议
1. {基于相关性的建议}
2. {需进一步验证的因果关系}
3. {可能需要控制的混淆变量}
```

## 约束

- 相关系数不等于因果关系，结论中必须注明"相关性不代表因果性"
- 样本量少于 10 时，注明"样本量不足，相关性结论可能不可靠"
- 不对非数值列计算相关性
- 发现强相关时，提示用户可能存在多重共线性问题
