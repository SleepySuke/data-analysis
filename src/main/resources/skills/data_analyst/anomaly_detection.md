# 异常值检测分析

## 目标

检测数据中的异常值和离群点，分析异常产生的原因，帮助用户识别数据质量问题或业务异常事件。

## 前置条件

- 数据必须包含至少 1 个数值型列
- 数值列的有效数据不少于 20 条（过少无法有效判定异常）
- 异常值检测基于统计方法（非机器学习）

## 执行步骤

### 步骤 1：数据概览

调用 `analyzeCsv` 工具，参数：
- `csvData`: 原始 CSV 数据
- `analysisGoal`: "异常值检测：了解数值列分布和极端值情况"

获取结果后关注：
- 识别所有数值列及其 min / max / mean
- 关注 mean 与 min/max 的差距，差距悬殊可能暗示异常值
- 检查 uniqueValues，唯一值过少（≤ 5）的列不适合做异常检测

### 步骤 2：统计计算

调用 `calculateStatistics` 工具，参数：
- `csvData`: 原始 CSV 数据
- `metrics`: "mean,median,stddev,skewness,kurtosis"

基于统计结果判断异常：
- **均值 vs 中位数**：差异超过 1 个标准差 → 存在偏移异常值
- **偏度（skewness）**：|偏度| > 2 → 数据严重偏斜，可能有极端值
- **峰度（kurtosis）**：峰度 > 7 → 尖峰分布，尾部有极端值
- **IQR 方法**：超出 Q1 - 1.5×IQR 或 Q3 + 1.5×IQR 的值为异常

### 步骤 3（可选）：知识检索

如果检测到的异常可能涉及行业特定原因，调用 `searchKnowledge` 工具：
- `goal`: "{行业}常见数据异常原因和处理方法"

## 输出格式

按以下结构输出：

```
【异常值检测报告】

一、检测范围
检测的数值列：{列名列表}
检测方法：IQR 方法 + 统计偏度/峰度分析

二、各列异常检测结果
{每个数值列}

列：{列名}
- 均值：{value}，中位数：{value}，标准差：{value}
- 偏度：{value}，峰度：{value}
- 异常判断：{是/否}
- 异常特征：{描述偏离情况}
- 推测原因：{基于数据特征的推测}

三、异常汇总
- 存在异常的列数：{n}/{total}
- 最严重的异常列：{列名}，原因：{描述}
- 整体数据质量评估：{良好/一般/较差}

四、处理建议
1. {对每类异常的处理建议}
2. {是否建议转交 data_cleaner 处理}
```

## 约束

- 样本量少于 20 时注明"样本不足，异常检测仅供参考"
- 不直接修改或删除数据，仅提供分析报告
- 明确标注"异常值"是统计意义上的异常，不一定是错误数据
- 如果多数数值列都有异常，建议用户转交 `data_cleaner` 做专业清洗
