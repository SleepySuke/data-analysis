# 销售趋势分析

## 目标

分析销售数据的趋势变化，识别增长和下降模式，找出关键转折点，并为未来走势提供判断依据。

## 前置条件

- 数据必须包含时间维度列（日期、月份、季度或年份）
- 数据必须包含至少一个数值型指标列（销售额、销量、利润等）
- 数据量建议不少于 12 个时间点以识别趋势

## 执行步骤

### 步骤 1：数据概览

调用 `analyzeCsv` 工具，参数：
- `csvData`: 用户提供的原始 CSV 数据
- `analysisGoal`: "销售趋势分析"

获取结果后关注：
- 确认时间维度列的名称和格式（检查 inferredType 为 date 的列）
- 确认数值指标列（inferredType 为 numeric 的列）
- 检查缺失率（missingRate），缺失超过 20% 的列需在分析结论中说明

### 步骤 2：统计计算

调用 `calculateStatistics` 工具，参数：
- `csvData`: 原始 CSV 数据
- `metrics`: "mean,median,stddev"

分析统计结果：
- 均值与中位数差异大 → 数据存在极端值偏移
- 标准差相对均值比例高 → 销售波动剧烈
- 比较不同指标的波动程度，识别最稳定的指标

### 步骤 3（可选）：知识检索

如果涉及特定行业的销售分析（如金融、医疗、零售），调用 `searchKnowledge` 工具：
- `goal`: "{行业}销售分析方法和基准数据"

将检索到的行业知识融入分析结论中作为参考。

### 步骤 4：可视化

调用 `generateChart` 工具，参数：
- `chartType`: 推荐使用 `line`（折线图）展示趋势
- `analysisResult`: 步骤 1-3 的分析摘要
- `csvData`: 原始数据

可选补充：
- 如需同比对比，额外生成 `bar`（柱状图）
- 如展示构成变化，可生成 `pie`（饼图）

## 输出格式

按以下结构输出：

```
【数据概要】
数据来源：{描述}
时间范围：{起止时间}
分析指标：{列出核心数值列}

【趋势分析】
整体趋势方向：{上升/下降/平稳/波动}
关键转折点：{列出时间点和可能原因}
异常波动：{描述偏离正常水平的时段}

【统计摘要】
{每个核心指标的均值、中位数、标准差}

【分析建议】
1. {基于数据的第一条建议}
2. {基于数据的第二条建议}
3. {基于数据的第三条建议}
```

## 约束

- 不要编造数据中不存在的趋势
- 如果数据不足（少于 6 个时间点），明确告知用户趋势分析不可靠，建议补充数据
- 图表必须基于实际数据生成，不可使用示例数据
- 缺失数据超过 30% 时，在结论首行标注数据完整性警告
