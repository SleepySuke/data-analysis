# 去重与标准化

## 目标

去除数据中的重复记录，执行类型转换和格式标准化，确保数据的一致性和规范性。

## 前置条件

- 数据为结构化格式
- 至少包含 1 行数据
- 用户需明确去重的判断依据（基于哪些列判断重复）

## 执行步骤

### 步骤 1：去重与标准化

调用 `executeScript` 工具，参数：
- `scriptPath`: "data_cleaner/scripts/dedup_standardizer.py"
- `arguments`: {"dedup_columns": "col1,col2", "dedup_strategy": "first", "standardize": "all"}
- `stdinData`: 原始 CSV 数据

处理内容：
- 完全重复的行去除
- 基于指定列的重复行去除
- 近似重复的行去除（大小写/空格差异）
- 重复记录的示例展示

去重策略说明：
- **完全去重**：所有列完全一致的行只保留第一条
- **关键列去重**：基于指定列（如 ID、订单号）判断重复
- **模糊去重**：考虑大小写、空格、标点差异

冲突处理策略：
- 保留第一条（默认）
- 保留最后一条
- 合并（数值列取均值）

格式标准化：
- **字符串标准化**：统一大小写、去除首尾空格、全角转半角
- **日期标准化**：统一为 `yyyy-MM-dd` 格式
- **数值标准化**：统一小数位数、去除千分位分隔符
- **枚举标准化**：统一同义词（如 "男"/"M"/"male" → "男"）

### 步骤 2：审阅结果

查看步骤 1 的输出，确认：
- 去重行数是否符合预期
- 标准化变更是否正确
- 数据完整性是否保持

## 输出格式

按以下结构输出：

```
【去重与标准化报告】

一、去重结果
原始行数：{n}
重复行数：{n}（{百分比}）
去重后行数：{n}
保留率：{百分比}

去重规则：
- 判断依据：{列名列表}
- 冲突策略：{保留第一条/最后一条/合并}

二、标准化结果
| 列名 | 标准化类型 | 变更数量 | 示例 |
|------|------------|----------|------|
| {日期列} | 日期格式统一 | {n} | {before} → {after} |
| {名称列} | 大小写统一 | {n} | {before} → {after} |

三、数据变更汇总
- 总变更数：{n}
- 影响列数：{n}
- 数据完整性：{变更后状态}

四、建议
{后续处理建议}
```

## 约束

- 去重操作不可逆，执行前必须确认
- 标准化规则需展示示例，确保用户理解变更内容
- 不改变数据的业务含义（如不将"未知"改为具体值）
- 保留原始数据的备份引用，便于核对
